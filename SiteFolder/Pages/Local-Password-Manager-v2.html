Local Password Manager V2



Goals

  1. Manage passwords in a secure way
  2. Be accessible everywhere (pc and phone)
  3. Sync information between devices safely
  4. No installation required

Manage passwords in a secure way

I use a simple script that encrypts data supposedly with AES256 encryption.

check the autor of the script below:

AES implementation in JavaScript © Chris Veness 2005-2011

  • see http://csrc.nist.gov/publications/PubsFIPS.html#197

  • code used here -> https://jonlabelle.com/snippets/view/javascript/javascript-aes-encryption

    I look at the script and it seemed safe and simple and did its job so I used it.

Improvements

Maybe use PGP encryption in the future or an updated encryption script

Be accessible everywhere (pc and phone) / No installation required

This point was essential for me because most password managers out there need some kind of installation (app or windows install) and some only work on PC, others are not free…

So what I wanted to do was a browser application that worked both on mobile and pc (actually any device with a browser) and it is free and does not need installation because works on a browser.

improvements

Update cache management to update the program when I make a git push.

Sync information between devices safely

One crucial feature to make this software usable is data synchronization across devices.

Nowadays this is possible through an external server and secret key

but in the future, I hope to make this data synchronization via peer-to-peer software like webRTC. Note here, data is always encrypted when is exchanged, of course!

Improvements

sync using peer-to-peer webRTC instead of an external server.
